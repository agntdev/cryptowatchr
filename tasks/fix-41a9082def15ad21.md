# fix-41a9082def15ad21 — Morning summary delivery scheduler never implemented

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 CWTR

E4T2 specifies implementing the morning summary feature where users can set a time for a daily summary of their watched coins. The configuration UI works: `/summary` command, `menu:settings` integration, enable/disable/time change buttons, time input parsing, and persistence (`setMorningSummary`/`getMorningSummary`/`deleteMorningSummary` in `src/store.ts`).

However, the actual delivery mechanism is completely missing. `src/poller.ts` only evaluates alert rules and delivers alert messages — there is no code that checks whether any user's configured summary time has arrived, fetches prices for their watchlist, formats a summary message, and sends it. Users can configure morning summaries but will never receive one.

Fix: add a summary delivery loop to `src/poller.ts` (or a separate scheduler) that periodically checks all users' configured summary times against the current time in their timezone, fetches watchlist prices, and sends the formatted summary via `sendMessage`.
